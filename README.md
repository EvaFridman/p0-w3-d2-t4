# p0-w3-d2-t4 · Валидация — на функции

> Тип: связка с вчерашним днём · Репозиторий: один отдельный

> Подключить файл, отсмотреть в браузере.

## Задача

Вчера ты проверяла данные регистрации «в лоб», одним длинным куском. Сегодня тот же смысл, но правильно: каждую проверку выносишь в отдельную маленькую функцию, которую можно вызвать где угодно и переиспользовать. Так валидация и пишется в реальных проектах.

```
валидацию из вчерашней задачи → в функции
  isFilled(value)  → true/false
  hasAtSign(email) → true/false
  isAdult(age)     → true/false
маленькие функции-проверки, каждую можно вызвать отдельно
```

## Что нужно сделать

В `validate.js` написать набор функций-проверок, каждая принимает значение и возвращает `true`/`false`: `isFilled(value)` (не пусто и не одни пробелы); `hasAtSign(email)`; `isNumber(value)` (именно число); `isAdult(age)` (число и в диапазоне). Затем взять несколько наборов данных пользователя и, вызывая эти функции, вывести вердикт по каждому.

## Критерии приёмки

* Каждая проверка — отдельная функция, возвращающая булево, вызывается по имени.
* Функции переиспользуются на нескольких разных наборах данных, не дублируя логику.
* `isNumber` отсекает строку `'25'`, `isFilled` — пустую строку и пробелы; ты можешь объяснить, где здесь важно строгое сравнение.
